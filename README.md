# G1 动作仿真（ラジオ体操路点验证）

在没有真机的情况下，可视化验证广播体操动作路点：像不像、对称吗、有没有自碰撞、
节奏合不合适。**今晚在这里把动作调到满意，明天真机时段只做"映射+确认"，不做设计。**

## 这套仿真能/不能验证什么（如实）

- ✅ 动作外观、左右对称性、自碰撞、每节动作时长
- ❌ 真机的平衡与动力学（这是纯运动学回放，机器人不会摔倒）。
  真机平衡靠机器人自带的平衡控制器 + 我们只做慢速对称小幅动作来保证，
  最终确认仍需明天真机时段完成。

## 安装（约5分钟，全程只需联网下载这两样）

```
pip install mujoco
```

下载G1官方仿真模型（来自Google DeepMind维护的mujoco_menagerie模型库）：
- 方式A：git clone https://github.com/google-deepmind/mujoco_menagerie
- 方式B：浏览器打开该仓库 → Code → Download ZIP → 解压
- 只需要其中的 `unitree_g1/` 文件夹

然后把 `sim_viewer.py` 顶部的 `MODEL_PATH` 改成你解压后的
`.../mujoco_menagerie/unitree_g1/scene.xml` 路径。

## 使用

```
python sim_viewer.py
```

- 数字键 **1-4** 切换动作（伸展/转体/上下伸举/深呼吸），默认每节只播一遍
- **R** 重播　**L** 切换单次/循环　**-**/**=** 减速/加速(0.25~2倍)　**空格** 暂停　**ESC** 退出
- 按键前先用鼠标点一下MuJoCo窗口获得焦点
- 出现自碰撞会在终端打印警告（哪两个部件、在第几秒）

调参循环：改 `motions.py` 里的角度数字 → 重跑 → 肉眼看效果。
方向反了就把正负号翻过来（不同模型版本关节符号约定可能不同，视觉迭代最快）。

## 定稿后

```
python export_waypoints.py
```

生成 `motions_export.md` 路点表，明天真机时段直接照表做arm_sdk映射，
配合《共享时段检查表》使用——这份表就是那张表里"本时段必须完成项"的输入。

## 常见问题

- **报"找不到匹配XX的关节"**：终端会打印模型里全部关节名，对照修改
  `motions.py` 里的关键词元组即可（23自由度和29自由度版本命名略有差异）。
- **窗口打不开/黑屏**：确认显卡驱动正常；`pip install mujoco` 自带渲染依赖，
  一般Windows上开箱即用。
- **想跑带物理的完整仿真**：宇树官方有 unitree_mujoco 仓库（同一套DDS接口，
  代码可直接迁移到真机），但偏Linux、搭建成本高，**今晚不建议碰**，
  运动学验证已覆盖我们最需要的部分。
